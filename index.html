VLAN Trunking Protocol (VTP)

VTP es un protocolo de mensajería de Capa 2 que mantiene la consistencia de la base de datos de VLANs sincronizando configuraciones a través de enlaces troncales.

Propagación automática de ID, nombres y parámetros de estado en un dominio administrativo.

• JHON ALEJANDRO CORREAL • RAFAEL ANDRÉS GUZMÁN • MARÍA HELENA MORENO • LAURA MARIANA MUÑOZ

Requisitos de Operación

01 Enlaces Trunk (IEEE 802.1Q) activos
02 Nombre de Dominio VTP idéntico
03 Password VTP (Case sensitive)
04 Versión de protocolo compatible (v1/v2/v3)

Propósitos del Protocolo

Consistencia

Asegura que todos los switches de la red reconozcan las mismas VLANs, evitando problemas de conectividad por falta de configuración local.

Administración

Centraliza la gestión lógica de la red. Una modificación en el switch Server se refleja automáticamente en todos los clientes.

Optimización

Reduce el overhead de configuración manual y permite el uso de VTP Pruning para gestionar el ancho de banda en los troncales.

Modos de Operación

ADMINISTRACIÓN / MASTER

Server

Puede crear, modificar y eliminar VLANs. Envía anuncios VTP cada 5 minutos o cuando ocurre un cambio. Guarda info en NVRAM.

REPLICACIÓN / SLAVE

Client

Escucha anuncios y actualiza su base de datos local. No permite cambios manuales de VLAN. No guarda info en NVRAM.

BYPASS / LOCAL

Transparent

No participa en el dominio de sincronización. Reenvía anuncios VTP pero mantiene su propia base de datos aislada en NVRAM.

Mensajes y Comunicación VTP

Summary Advertisements

Contienen el nombre del dominio, el número de revisión y el timestamp. Se envían para informar a los switches vecinos del estado actual.

Subset Advertisements

Contienen la información detallada de las VLANs (ID, nombre, MTU). Se envían inmediatamente después de un cambio de configuración.

Advertisement Requests

Enviados por un cliente cuando detecta que su número de revisión es menor al del servidor o cuando se reinicia el switch.

Mecanismo de Sincronización

Elemento crítico de control:

CONFIGURATION REVISION NUMBER 0x00000C

Es un número de 32 bits que indica el nivel de revisión de una configuración VTP. Si un switch recibe un número de revisión superior, asume que su base de datos es antigua y sincroniza.

Análisis de Implementación

Ventajas Operativas

  • Seguimiento preciso de VLANs en toda la organización.
  • Configuración de VLANs dinámicas (Plug-and-play).
  • Soporte para VTP Pruning (Optimización de inundación).
  • Facilidad de monitoreo de red centralizado.

Desventajas y Riesgos

  • VTP Bombing: Un switch con revisión alta puede borrar la red.
  • Propagación instantánea de errores tipográficos.
  • Propiedad de Cisco (No es un estándar abierto).
  • Limitado a la base de datos de VLAN estándar (1-1005).

Configuración en CLI (Switch PEPE)

# Definición de Hostname y Seguridad Switch# configure terminal
PEPE(config)# hostname PEPE
PEPE(config)# enable secret pepe
# Configuración del Protocolo VTP PEPE(config)# vtp domain ETITC_LAB
PEPE(config)# vtp mode server
PEPE(config)# vtp password secure123
# Creación de VLANs PEPE(config)# vlan 10
PEPE(config-vlan)# name distribucion
PEPE(config-vlan)# vlan 20
PEPE(config-vlan)# name registro
# Configuración de Troncal e interfaces de Acceso PEPE(config)# interface fastethernet 0/1
PEPE(config-if)# switchport mode trunk
PEPE(config-if)# exit
PEPE(config)# interface range fastethernet 0/2-3
PEPE(config-if-range)# switchport mode access
# Verificación del estado del dominio PEPE# show vtp status
PEPE# write memory
