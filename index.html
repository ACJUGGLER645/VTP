VLAN Trunking Protocol

VTP Infraestructura Conmutada

Protocolo de capa 2 utilizado principalmente en switches Cisco para la gestión centralizada y automática de VLANs. Permite propagar ID, nombre y estado entre dispositivos de un mismo dominio.

Funcionamiento de VTP

REQ Utiliza anuncios VTP periódicos
REQ Requiere un dominio VTP común
REQ Opera sobre enlaces trunk activos
REQ Emplea número de revisión de cambios

Propósitos del Protocolo

Consistencia

Garantizar la consistencia de la base de datos de VLAN en múltiples switches de la red local.

Administración

Centralizar la administración de VLAN y reducir los errores humanos derivados de la configuración manual.

Optimización

Optimizar el tiempo de despliegue de la red al automatizar la propagación de cambios estructurales.

Modos de Operación

ADMINISTRACIÓN / MASTER

Server

Administra y propaga VLANs. Puede crear, modificar y eliminar la base de datos de VLAN en todo el dominio.

REPLICACIÓN / SLAVE

Client

Replica la base de datos de VLAN recibida del servidor. No permite realizar cambios locales en la configuración.

BYPASS / LOCAL

Transparent

Reenvía anuncios VTP por sus enlaces troncales pero no aplica cambios a su propia base de datos local.

Control de Cambios

Indicador de versión de base de datos:

CONFIGURATION REVISION NUMBER 0x00000C

Al detectar un anuncio con un número de revisión superior al local, el switch sincroniza su base de datos para igualar el estado del servidor VTP.

VTP vs. Configuración Manual

Ventajas VTP

  • Administración centralizada y escalable.
  • Menor probabilidad de inconsistencia de datos.
  • Simplifica la gestión en redes de gran escala.
  • Reduce el trabajo repetitivo del administrador.

Configuración Manual / Riesgos

  • VTP Bombing: Riesgo de borrado accidental de VLANs.
  • Propagación masiva de errores de configuración.
  • La configuración manual ofrece mayor control individual.
  • Hoy se prefiere modo transparente por seguridad.

Laboratorio de Configuración (Switch)

Cisco IOS - Configuración VTP Completa
# 1. Seguridad Básica y Hostname Switch# configure terminal
PEPE(config)# hostname PEPE
PEPE(config)# enable secret pepe
# 2. Configuración de Red VTP ! Observación: El dominio y password deben ser idénticos en todos los switches para sincronizar PEPE(config)# vtp domain ETITC_LAB
! Observación: Modo Server (Default) permite crear VLANs y aumenta el 'Revision Number' PEPE(config)# vtp mode server
PEPE(config)# vtp password cisco123
# 3. Creación de Datos (VLANs) PEPE(config)# vlan 10
PEPE(config-vlan)# name distribucion
PEPE(config-vlan)# vlan 20
PEPE(config-vlan)# name registro
# 4. Configuración de Puertos de Acceso PEPE(config)# interface fastethernet 0/3
PEPE(config-if)# switchport mode access
PEPE(config-if)# switchport access vlan 10

PEPE(config)# interface fastethernet 0/2
PEPE(config-if)# switchport mode access
PEPE(config-if)# switchport access vlan 20
# 5. Configuración Troncal (Vital para el transporte de anuncios VTP) PEPE(config)# interface fastethernet 0/1
PEPE(config-if)# switchport mode trunk
PEPE(config-if)# end
# 6. Verificación de Estado (show vtp status) PEPE# show vtp status

Configuración de Modos Cliente y Transparente

Switch_Cliente (Sincronizado)
# Configuración para recibir actualizaciones Switch(config)# vtp domain ETITC_LAB
Switch(config)# vtp password cisco123
Switch(config)# vtp mode client
Setting device to VTP CLIENT mode.
! Observación: Si intentas crear una VLAN en este modo, el IOS dará error: Switch(config)# vlan 30
VTP VLAN configuration not allowed when device is in CLIENT mode.
Switch_Transparente (Aislado)
# Configuración para ignorar sincronización Switch(config)# vtp domain ETITC_LAB
Switch(config)# vtp password cisco123
Switch(config)# vtp mode transparent
Setting device to VTP TRANSPARENT mode.
! Observación: En este modo el 'Revision Number' siempre será 0 Switch# show vtp status | include Revision
Configuration Revision : 0
# Permite crear VLANs locales que no se propagan Switch(config)# vlan 99
Switch(config-vlan)# name vlan_privada
